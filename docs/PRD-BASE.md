# 1. 기초요구사항 확인 및 상세요구사항 질의서 생성

언어학습 서비스인 FlipFlip 서비스를 개발하려고 합니다.
아래와 같이 기본적인 서비스 개요을 참고하여 상세서비스 요구사항을 정의하기 위한 절차를 진행해주세요.

분석가 서브에이전트를 활용하여 요구사항 수집을 위한 질의서를 만들어주십시오. 질의서는 단계별, 시스템별 등 나누어서 만들어도 됩니다.
질의서에 답하면 해당 답변을 기반으로 이후에 상세설계를 진행하면 됩니다.

<기본서비스개요>

## 1. 배경

- 너무 복잡한 영어공부. 그냥 심플하게 매일 따라만 해도 되는 저렴한 서비스가 있었으면 함
- 언어학습에 있어서 반복학습이 중요한데 경쟁사의 앱은 문법이나 너무 복잡하고 많은 콘텐츠를 담고 있어서 학습에 오히려 부담이 됨.
- 카드를 넘기듯이 모국어 의미를 확인하고 학습언어를 반복연습하는 쉬운 UI를 통해 언어를 문장 단위로 학습할 수 있음.
- 컨셉 : 세상에서 가장쉬운 언어학습앱

## 2. 시스템 개요

- 주요타겟 : 중등교육을 받아 영어에 대해서 기본적인 이해가 있는 사람으로 언어학습에 관심이 있는 사람.
- 시스템 : 관리자웹 (React, TailwindCSS),사용자용웹 (React, TailwindCSS), API, 사용자용 모바일앱 (ReactNative, Expo)

## 3. 서비스 핵심특장점

- 심플한 UI : 카드를 넘기는 형태의 메인 학습UI는 손쉬운 사용성 제공
- 체계화된 학습 : 레벨 - 스킬 - 문장 - 표현
  - 레벨 : 1~100레벨까지 하위에 5개 내외의 스킬로 구성. 원어민 기준 수준
  - 스킬 : 주요단어, 시제, 문형, 표현 등 언어 스킬로 한 레벨에 5개 내외의 스킬이 포함됨
  - 문장 : 스킬을 학습할 수 있는 문장으로 관리기능에서 의미를 관리하기 위한 것임. 의미는 국문, 영문 2가지 버전으로 담고 있으며, 국문은 한국어를 모국어로 선택한 관리자에게 표시되며, 영문의미는 한국어 이외의 언어를 모국어로 설정한 관리자에게 표시됨.
  - 표현 : 의미에 대응되는 언어별(표현언어) 표현으로, 학습 시 해당 문장의 "의미"를 표현함에 있어 모국어로 선택한 언어가 표현언어에 없을 경우 영어가 기본으로 표시됨.
  - 표현언어 : 초기 표현언어는 영어, 스페인어, 중국어, 일어 (4개 언어)
  - 학습언어 : 영어, 스페인어, 중국어, 일어, 한국어 (5개 언어)
  - 모국어 : 한국어, 영어 (2개 옵션, 추후 확장 가능)
- 기억 최적화 : 망각곡선 이론을 반영하여 학습빈도에 따라 다음 학습시점을 조정하여 최적의 시간에 최대한 잘 기억하도록 함
- 상황 인지 : 문장에 대한 빠른 이해와 기억력을 높이기 위해 시제 (현재, 과거, 과거완료, 미래 등), 문장용도 (질문, 요구, 설득, 설명 등), 톤 (비공식, 공식), 장소 (회사, 집, 여행 등)등 상황표시 태그가 아이콘, 색상 등을 활용하여 표시되어 상황정보를 전달함.
- 문장용도별 레이더차트형태의 통계를 제공하여 사용자가 현재 어떤 부분이 약한지 확인하고, 자신의 수준이 원어민 수준으로 어느 정도 되는지 알게 함으로써 자신감을 가질 수 있게 함. 언어는 기세.

## 4. 주요서비스 프로세스 (사용자웹) 및 정책

- 온보딩 : 서비스 특장점에 대한 콘텐츠를 확인. 온보딩 마지막에 시작하기 버튼 선택하면 온보딩 조회플래그 true로 로컬에 저장되어 다음에는 건너뜀
- 인증 : 애플인증(모바일앱에서만 제공), 구글인증 2가지 제공
- 회원가입
  - 인증 후 약관동의 (국가에 맞는 서비스이용약관, 개인정보 처리방침, 개인정보수집 및 이용동의, 유료서비스 이용약관, 마케팅 활용동의)
  - 약관동의 후 전화번호 인증 (필수), 닉네임 (필수, 중복체크, 로그인 사용자이름 기본입력), 공유용 이메일 (인증이메일 기본입력), 생년월일 (필수, 성인만 이용가능), 프로필 이미지를 업데이트하면 회원가입이 완료됨. (인증용 이메일은 별도로 저장됨)
  - 회원가입 마지막 단계에 추천인ID입력하고, 아이디가 확인되면
- 레벨테스트
  - 사용자의 수준을 확인하고 수준에 맞는 레벨로 이동하기 위해 테스트 진행.
  - 모국어와 학습언어를 선택 후 레벨별 의미를 표시하고, 맞추면 10단계 위의 문제를 출제하고 틀리면 이전에 맞춘 가장 높은 단계와 틀린 단계의 중간의 단계 문제를 출력하는 방식으로 레벨을 선택함.
  - 이때 틀린단계/맞춘단계를 정하는 것은 한문제로 하는 것이 아니라 3문제를 내서 2문제 이상 맞춘 경우를 기준으로 하며, 문장에 대한 맞춤의 의미는 완벽히 맞춘 것이 아니라 대부분 맞춤까지도 포함한다.
  - 테스트는 처음만 하는 것이 아니라 학습 중간에도 할 수 있으며 이 경우 테스트 시작레벨은 현재레벨로 함.
- 일학습목표시간 설정
  - 레벨테스트 완료 후, 학습 시작 전에 사용자가 일학습목표시간을 선택함 (5분, 15분, 30분, 1시간, 2시간, 2시간이상)
  - 하루 학습완료 기준 : 선택한 분 × 1분당 1문장 = 최소 완료 문장 수 (예: 15분 선택 시 최소 15문장 완료)
  - 일학습목표시간은 유형관리에서 "일학습목표시간" type으로 관리됨
  - 설정 후 변경 가능 (마이페이지 또는 학습설정에서)
- 학습
  - 테스트가 완료되면 레벨이 설정되며, 설정된 레벨의 학습데이터를 다운로드 받아서 학습이 진행됨.
  - 학습데이터
    - 새학습내용
      - 최초 : 시작레벨의 첫번째 스킬에 포함된 문장 전체
      - 스킬업 시 : 스킬업된 스킬에 포함된 문장 전체
      - 레벨업 시 : 업된 레벨의 첫번째 스킬에 포함된 문장 전체
    - 기존학습 내용 : 학습할 데이터는 다음학습시간이 도래한 모든 문장 중 랜덤하게 골라서 나옴 (이때 이전에 학습한 문장의 )
    - 다음학습시간 : 익힘수준에 따라 다음학습시간이 결정됨 (익힘수준유형 및 수준별 학습시간은 유형관리에서 관리)
  - 체크
    - 표시된 의미문장을 보고 학습언어로 문장을 추측해본 후 정답을 맞춰보는 과정을 "체크"라고 함.
    - 체크 유형은 "완벽히 맞춤", "대부분 맞춤", "못 맞춤" 3가지임. 체크점수는 완벽히 맞춤 2점, 대부분 맞춤은 0.5점, 못 맞춤은 0점 (유형관리에서 유형 및 점수가 관리됨. 점수값은 value값)
    - 컨트롤 : 화면에 표시된 키보드를 이용해서 조작가능함. "완벽히 맞춤"은 up, "대부분 맞춤"은 left, "못 맞춤"은 right, 정답보기는 "down" 키로 동작함.
    - 익힘수준 및 익힘점수
      - 문장에 대한 학습정도를 말하는 것으로 "완벽", "매우익숙", "익숙", "미비", "첫암기"(예를 들어 "매우익숙"의 기준점수는 5점이며, 다음학습간격은 7일 x 24 x 60, "익숙"은 3점이며 반복기한은 1일 x 24 x 60분, 유형관리에서 관리, 점수는 value, 다음학습시간은 부가정보1에 분단위로 정수값입력)
      - 체크 시 체크점수를 누적하고, 누적된 체크점수로 익힘수준이 업데이트 되며, 익힘수준의 기준점수가 익힘점수가 됨.
    - 스킬업
      - 현재 공부하는 스킬에 포함된 문장을 잘 학습했는지 정도를 판단하는 것으로 학습성취점수가 학습총점수의 일정비율(정책관이에서 입력된 비율)이상이 되면 스킬업됨 (스킬 게이지가 학습화면 상단에 표시됨)
      - 학습성취점수 계산 : 해당 스킬에 포함된 문장의 익힘점수의 합
      - 학습총점수 계산 : 해당 스킬에 포함된 문장수 x "완벽" 기준점수
      - 클리어 : 스킬업되는 것을 해당 스킬이 클리어 된 것으로 봄
      - 스킬업 메시지 : 해당 스킬이 클리어되면 환영 팝업 표시
    - 레벨업
      - 현재 레벨에 포함된 모든 스킬이 클리어(스킬업)되면 해당 레벨이 클리어 되어 다음 레벨로 레벨업됨.
      - 레벨업 메시지 : 해당 레벨이 클리어 되면 레벨업 환영 팝업 표시
    - 연속학습 : 학습습관을 만드는 것이 중요하기 때문에 학습화면 상단에는 캘린더 컴포넌트 제공. 미완료한날과 완료한날, 미래날짜 구분되며 이번주 (일~ 토요일) 한 주가 표시됨
- 기록기능
  - 캘린더 컴포넌트 월단위 모드로 연속학습 표시
- 마이페이지 : 프로필 업데이트, 현재플랜 및 유료화결제기능, 로그아웃기능
- 유료화
  - 30일간 무료사용 가능
  - 30일 이후 유료전환 필요. 안내 표시. 유료화 미전환 시 학습기능 제한 (하루 3문장만 학습가능). 데이터는 그대로 저장됨.
  - 유효화는 1개월기준 한국 990원, 해외 $0.99, 1년 기준 9900원, 해외 $9.9
  - 결제 : 모바일은 인앱결제 + 외부온라인 결제 2가지 제공, PC웹은 온라인결제 (국내유명 PG사 + 해외는 해외PG사)
- 친구초대 기능
  - 가입 시 추천인 ID입력방식
  - 친구가입 완료하면 일주일 무료쿠폰 지급
- 공유기능
  - 레벨업 시 공유기능제공 (레벨업 이미지, 레벨, 레벨개요)
- 마일리지
  - 축적 : 하루 학습완료 시 10pt, 3일연속 학습시 연속학습포인트 30pt 추가지급, 10일 연속학습 성공 시 200pt 추가지급, 30일 연속학습 시 500pt 추가지급
  - 사용 : 10,000pt로 일주일학습권 교환가능 (해당쿠폰 사용시)

## 5. 관리기능

- 로그인 : 구글 OAuth로그인이며, help@cookieapps.kr 이메일이 아닌 경우 로그인 불가 안내. 이름/프로필 이미지는 구글정보 활용.
- 유형관리 : 다단계 유형관리, 유형정보 (유형명, 유형코드 - 중복체크기능, 표시순번, 값, 요약, 부가속성1 ~ 5, 사용여부)
- 콘텐츠관리
  - 화면은 세로 4단으로 구성됨 (레벨컬럼, 스킬컬럼, 문장컬럼, 표현컬럼)
  - 레벨은 1 ~ 100레벨까지 고정값으로 표시되며, 레벨(레벨번호, 레벨개요)을 선택하면 우측 스킬컬럼에 해당레벨 하위 스킬목록(스킬명, 스킬개요)이 표시됨.
  - 스킬항목을 선택하면 스킬에 포함된 (해당 스킬ID가 있는) 문장목록이 표시됨
  - 문장목록을 선택하면 문장에 포함된 (해당 문장ID가 있는) 표현목록이 표시됨
  - 스킬, 문장, 표현 컬럼의 컬럼헤더에는 항목추가버튼 표시되며, 추가버튼 누르면 항목추가/수정 팝업 표시
  - 스킬, 문장, 표현 각 항목은 선택 시 항목수정 팝업이 표시되며, 수정팝업에는 삭제, 수정기능 제공
  - 정보구조
    - 레벨 : 레벨번호(정수, 1~100까지 고정), 레벨개요 (100글자), 원어민 수준유형값
    - 스킬 : 레벨ID, 스킬명, 스킬개요 (200글자), 정렬순서 (정수), 시제유형, 용도유형, 톤유형, 장소유형
    - 문장 : 스킬ID, 문장개요 (200글자)
    - 표현 : 문장ID, 언어, 언어별표헌 (200글자), 힌트 (100글자)
- 레벨관리 : 1~100레벨까지 레벨정보 관리. 레벨정보 (레벨번호 - 고정, 레벨개요, 레벨설명), 좌측에 1~100까지 레벨목록이 있으며, 목록을 선택하면 우측에 레벨상세정보 표시되어 수정가능함. 하단에는 레벨에 포함된 스킬목록이 표시됨. 스킬을 선택하면 스킬수정팝업 표시.
- 정책관리
  - 레벨업 비율,
  - 스킬업 비율 관리
  - 마일리지 정책관리 : 친구초대 마일리지 / 무료이용가능일당 마일리지
- 회원관리 : 회원목록 (NO, 유/무료구분 - 배지, 닉네임, 이메일, ), 검색기능 (닉네임, 전화번호, 이메일 like 검색),
- 약관관리 : 국가별 약관유형별 콘텐츠 관리. 약관정보 (국가선택, 버전, 개정개요, 약관요약 200글자, 비합의시 안내, 사용여부, 필수동의 필요여부, 콘텐츠 - 콘텐츠는 버전별로 관리되며, 서식적용할 수 있어야 함)

## 6. 콘텐츠 제작 및 입력

- 약관 제작
  - 약관유형별로 한국 실정법에 맞는 약관콘텐츠 초안작성 및 DB입력쿼리 작성. 약관조항제목은 bold체가 되어야 함.
  - 기본정보 : 회사명 "쿠키앱스" / 담당자명 김무현 / 이메일 help@cookieapps.kr / 이외 입력해야 하는 정보는 "<OOO입력필요>"와 같이 더미정보로 생성
- 학습콘텐츠 정보 제작 : ./CONTENT-BASE.md 파일의 내용을 참고해서 생성하며 쿼리문으로 생성

## 7. 제약사항

- 로딩처리 : 학습데이터를 가지고 오는 것과 같이 데이터양이 많은 경우에는 progress bar를 이용해서 로딩처리하며, 이 경우 임의의 비율이 아닌 추정된 데이터양을 기준으로 진척이 반영됨.
- 로컬DB우선전략 : 사용성을 위해 로컬DB에 업데이트가 되면 UI가 업데이트 되며, 이후 원격DB와 싱크를 맞춤.
- 원격DB접속 효율화 : 데이터 처리비용을 최적화 하기 위해서 레벨업, 스킬업 시점에 학습문장세트를 한번에 받아오고, 학습데이터는 로컬DB에 저장하며, 당일 학습이 종료되는 시점에 학습기록을 원격DB에 저장하여 원격DB입출력을 효율화해야 함.
- 다국어 : 다국어지원 필요, 최초 영문, 국문 2가지만 제공. 설정에서 모국어 설정.
- 컴포넌트 기반 설계/제작
  - 중복되는 항목은 반드시 컴포넌트화하여 활용함.
  - 컴포넌트 유형
    - 코어컴포넌트 : input,checkbox, button 등
    - 콘텐츠 컴포넌트 : 목록을 구성하는 항목 등 기능별 컴포넌트는 해당 기능 하위에 두되, 여러 기능에 사용되는 항목은 common 아래에 위치
- 테이블 컬럼구성 : 이메일 등 값이 긴 컬럼만 가변으로 하고, 나머지는 값의 길이에 맞게 컬럼너비 고정값
