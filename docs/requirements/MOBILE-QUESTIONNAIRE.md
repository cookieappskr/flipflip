# FlipFlip 모바일앱 개발 질의서

> **작성일**: 2026-03-18
> **목적**: React Native + Expo 기반 모바일앱(iOS/Android) 개발 착수 전 확인이 필요한 사항
> **기준 문서**: PRD v1.0, Phase 3 (4주)

---

## 1. 기능 범위 (Scope)

### Q1-1. 웹 기능 대비 모바일 포함/제외 범위

현재 사용자 웹에 구현된 기능 중, 모바일 1차 출시에 포함할 기능과 제외할 기능을 확인합니다.

| 기능                            | 웹 현황 | 모바일 포함?         |
| ------------------------------- | ------- | -------------------- |
| Google 로그인                   | ✅      | ?                    |
| Apple 로그인                    | ❌      | ? (PRD에 명시됨)     |
| 회원가입 (7단계)                | ✅      | ?                    |
| 진단 (모국어/학습언어/목표시간) | ✅      | ?                    |
| 레벨테스트                      | ✅      | ?                    |
| 학습 (카드 플립)                | ✅      | ?                    |
| 기록 (월별 캘린더)              | ✅      | ?                    |
| 마이페이지                      | ✅      | ?                    |
| 유료화/결제 페이지              | ✅      | ? (인앱결제로 전환?) |
| 온보딩 (기능 안내)              | ✅      | ?                    |

**질문**: 1차 출시에서 제외할 기능이 있습니까? 또는 모바일 전용으로 추가할 기능이 있습니까?

제외할 기능은 없으며, ios출시를 위해서는 애플로그인이 필요하다고 하여 적용하려고 합니다. 이 경우 애플계정으로 로그인한 사용자가 웹에서는 로그인이 불가능할 것 같은데, "애플로그인을 이용해서 로그인한분은 앱으로 이용해주세요" 와 같이 웹에서 표시하는 것으로 하면 될까요?
그리고 결제는 인앱결제를 기본으로 하며 웹결제도 동일한 가격으로 하되 웹결제 링크도 같이 제공합니다.

---

### Q1-2. 푸시 알림 (Push Notification)

PRD에 푸시 알림 관련 명세가 없습니다.

- 학습 리마인더 푸시 알림이 필요합니까? (예: "오늘 아직 학습을 시작하지 않았어요")
- 필요하다면, 어떤 시점에 발송합니까?
  - [v] 일학습목표 미달성 시 (몇 시에?) => 사용자환경별 23:00에
  - [ ] 복습 문장이 있을 때
  - [ ] 연속학습 유지 격려
  - [ ] 기타: \_\_\_
- 사용자가 알림을 끌 수 있어야 합니까? => 네. 끌 수 있어야 합니다.

---

### Q1-3. 오프라인 학습

웹에서는 Dexie.js(IndexedDB) 기반 Local DB First를 구현했습니다. 모바일에서는 PRD에 WatermelonDB(SQLite)가 명시되어 있습니다.

- 모바일에서 **완전 오프라인 학습**을 지원합니까? (인터넷 없이도 학습 가능)
- 아니면 **약한 오프라인** (한번 다운로드 후 세션 중에만 오프라인)?

## 2. 인증 (Authentication)

### Q2-1. Apple Sign-In

PRD Phase 3에 Apple Sign-In이 명시되어 있습니다.

- iOS는 Apple Sign-In **필수**입니까? (App Store 정책상, 소셜 로그인을 제공하면 Apple Sign-In도 반드시 제공해야 함 → 필수)
- Android에서도 Apple Sign-In을 제공합니까, Google 로그인만 제공합니까?

**제안**: iOS = Apple + Google, Android = Google만 제공

네 제안대로 해주세요.

### Q2-2. 생체 인증 (Biometric)

- Face ID / Touch ID / 지문 인식 등 생체 인증으로 간편 로그인을 지원합니까?
- 지원한다면, 기존 로그인 후 옵션으로 활성화하는 방식입니까?

### Q2-3. 세션 유지

- 웹과 동일하게 Supabase Auth 세션을 사용합니까?
- 앱 재시작 시 자동 로그인이 유지되어야 합니까? (보통 모바일은 유지)
- 세션 만료 기간은? (웹 기본: Supabase default 1시간 + 리프레시 토큰)

- 아니면 웹결제(PG)도 병행합니까? (앱 내에서 웹뷰로 결제 등)

| ------- | ------- | --------- | -------------------- |
| 웹      | KRW 990 | KRW 9,900 | PG 수수료 ~3%        |
| iOS     | ?       | ?         | Apple 수수료 30%     |
| Android | ?       | ?         | Google 수수료 15~30% |

### Q3-2. 구독 복원

- "구독 복원" 기능이 필요합니까? (기기 변경 시 기존 구독 복원)
- 웹에서 결제한 구독을 모바일에서도 인정합니까? (cross-platform 구독)

## 4. UI/UX

### Q4-1. 디자인 기준

- 웹 UI를 그대로 모바일에 적용합니까, 별도의 모바일 디자인이 있습니까?
- Figma 등 별도 디자인 시안이 있습니까?
- 없다면, 웹 UI를 기반으로 모바일 최적화(네이티브 네비게이션, 탭바 등)하여 개발해도 됩니까?

### Q4-2. 제스처 (Gesture)

PRD의 체크 유형 키보드 단축키:

| 체크 유형  | 웹 키보드 | 모바일 제스처    |
| ---------- | --------- | ---------------- |
| 정답보기   | Down(↓)   | 탭 (카드 뒤집기) |
| 완벽히맞춤 | Up(↑)     | ?                |
| 대부분맞춤 | Left(←)   | ?                |
| 못맞춤     | Right(→)  | ?                |

현재 웹에서는 스와이프를 다르게 매핑했습니다 (오른쪽=완벽, 아래=대부분, 왼쪽=못맞춤).

**질문**: 모바일 스와이프 방향을 어떻게 매핑합니까?

- **안 A** (현재 웹): 오른쪽=완벽, 아래=대부분, 왼쪽=못맞춤
- **안 B** (PRD 기준): 위=완벽, 왼쪽=대부분, 오른쪽=못맞춤
- **안 C**: 스와이프 대신 버튼만 사용

### Q4-3. 네비게이션

- 하단 탭바 구성을 확인합니다. 웹 기준으로 아래와 같이 구성해도 됩니까?

## 5. 기술 (Technical)

### Q5-1. Expo 관리형 vs Bare

- **Expo Managed Workflow**로 개발합니까? (빌드/배포 간편, 네이티브 모듈 제한)
- **Expo Bare Workflow** 또는 **Expo Dev Client**를 사용합니까? (네이티브 모듈 자유, 빌드 복잡)

**제안**: Managed + EAS Build (인앱결제 등 네이티브 기능은 Expo Plugin으로 대응 가능)

### Q5-2. 최소 지원 OS 버전

PRD에 명시된 내용:

- iOS 16+
- Android 12+

이대로 진행합니까?

### Q5-3. 로컬 DB

PRD에 WatermelonDB가 명시되어 있습니다.

- WatermelonDB를 그대로 사용합니까?
- 또는 더 가벼운 대안 (expo-sqlite, MMKV 등)을 고려합니까?
- 동기화 전략은 웹과 동일하게 Last Write Wins입니까?

### Q5-4. 코드 공유

웹과 모바일 간 공유 가능한 코드:

- `scoring.ts` (점수 계산 로직)
- `database.ts` (타입 정의)
- `localDb.ts` (동기화 관련 타입)

이 코드들을 `packages/shared-utils/`로 추출하여 공유합니까, 아니면 복사하여 사용합니까?
