> **대상**: CookieApps 경영진 / IP 전략팀
> **문서 유형**: 특허 출원 가능성 사전 검토 의견서 (Patentability Assessment)

## 1. 검토 범위 (Scope of Review)

본 리뷰는 FlipFlip 서비스의 PRD 및 설계 문서를 기반으로 다음 4가지 발명 후보에 대해 **특허 출원 가능성(Patentability)**을 사전 검토합니다.

| No | 발명 후보 | 분류 |
|----|----------|------|
| 1 | 하이브리드 스킬-유형 시스템 (Hybrid Skill-Type System) | 데이터 구조 / 콘텐츠 관리 |
| 2 | 커스텀 망각곡선 알고리즘 (Custom Forgetting Curve Algorithm) | 학습 알고리즘 |
| 3 | 가중 랜덤 응용문장 시스템 (Weighted Random Application Sentences) | 학습 알고리즘 |
| 4 | 레이더차트 숙련도 분석 (Radar Chart Proficiency Analysis) | 데이터 시각화 / 분석 |

각 발명 후보에 대해 **신규성(Novelty)**, **비자명성(Non-Obviousness)**, **선행기술 리스크(Prior Art Risks)**, **출원 권고(Recommendation)**를 평가합니다.

## 2. 발명 후보 1: 하이브리드 스킬-유형 시스템

### 2.1 발명 개요

언어학습 콘텐츠의 메타데이터를 **두 개의 계층(스킬 레벨 / 문장 레벨)에 분산 배치**하는 하이브리드 관리 방식입니다.

```
[스킬 레벨 유형 관리]              [문장 레벨 유형 관리]
  - 시제 (Tense): 현재/과거/미래...    - 용도 (Purpose): 질문/요구/설명...
  - 문형 (Pattern): SV/SVO/의문문...   - 톤 (Tone): 공식/비공식
                                       - 상황 (Situation): 회사/집/여행...
```

핵심 차별점: 시제/문형은 **스킬 전체에 공통 적용**되는 구조적 특성이므로 스킬 레벨에서, 용도/톤/상황은 **문장마다 다를 수 있는** 맥락적 특성이므로 문장 레벨에서 관리합니다. 이 이원적 배치를 통해 콘텐츠 관리 효율성과 학습 분석 다차원성을 동시에 달성합니다.

### 2.2 특허성 평가

**신규성 (Novelty): 중간**

- 기존 언어학습 시스템에서도 문법(Grammar)과 상황(Context)을 분리하여 관리하는 사례가 존재합니다.
- 그러나 "스킬 계층"과 "문장 계층"이라는 **명시적 이원 구조**에서 유형을 분산 배치하는 방식은 기존 특허에서 직접적으로 확인되지 않습니다.
- Duolingo의 스킬 트리 구조는 스킬 단위로만 유형을 관리하며, 문장 단위 유형 배치는 별도로 하지 않습니다.

**비자명성 (Non-Obviousness): 낮음~중간**

- 데이터베이스 설계에서 "공통 속성은 상위 엔티티, 개별 속성은 하위 엔티티"라는 원칙은 일반적인 정규화(Normalization) 기법입니다.
- 이를 언어학습 도메인에 적용한 것이 "자명하지 않은(Non-Obvious)" 기술적 진보인지 논쟁의 여지가 있습니다.
- 특허 심사관이 "단순한 데이터베이스 설계 결정"으로 볼 가능성이 있습니다.

**선행기술 리스크 (Prior Art)**

| 선행기술 | 유사성 | 차이점 |
|---------|--------|--------|
| Duolingo 스킬 트리 (US Patent 9,818,312) | 스킬 기반 학습 구조 | 문장 레벨 유형 분리 없음 |
| Rosetta Stone 학습 구조 | 문법/상황 분류 | 명시적 이원 계층 아님 |
| CEFR 프레임워크 | 다차원 언어 능력 평가 | 특허 대상이 아닌 교육 프레임워크 |
| 다양한 LMS 특허 | 콘텐츠 메타데이터 계층화 | 언어학습 특화 적용이 아님 |

### 2.3 권고: **추가 조사 필요 (Needs Further Research)**

- 클레임(Claim)을 "하이브리드 유형 배치"의 기술적 효과(학습 분석 다차원성, 콘텐츠 관리 효율)에 초점을 맞추면 등록 가능성이 있습니다.
- 단, 순수 데이터 구조 특허는 미국(USPTO)에서는 Alice 판결 이후 소프트웨어 특허 심사가 엄격해졌으며, 한국(KIPO)에서도 "기술적 사상"으로 인정받아야 합니다.
- **비용 대비 효과**: 선행기술 조사(Freedom-to-Operate) 비용 약 200~500만원 소요 예상. 출원 전 정밀 조사 권고.

## 3. 발명 후보 2: 커스텀 망각곡선 알고리즘

### 3.1 발명 개요

에빙하우스(Ebbinghaus) 망각곡선 이론을 기반으로, **유형관리 시스템에서 설정 가능한 파라미터**를 통해 간격 반복(Spaced Repetition) 스케줄을 동적으로 제어하는 시스템입니다.

```
[익힘수준 체계]
  첫암기 (0점) → 30분 후 복습
  미비   (1점) → 4시간 후 복습
  익숙   (3점) → 1일 후 복습
  매우익숙 (5점) → 7일 후 복습
  완벽   (8점) → 30일 후 복습
```

핵심 차별점:
1. 익힘수준과 복습 간격이 **관리자 UI(유형관리)**에서 실시간으로 설정/변경 가능
2. 체크 유형별 점수(+2, +0.5, -1)도 유형관리에서 동적 조절 가능
3. 5단계 익힘수준 체계와 누적 점수 기반 수준 판정의 결합

### 3.2 특허성 평가

**신규성 (Novelty): 낮음**

- 간격 반복(Spaced Repetition)은 1880년대 에빙하우스의 연구에서 시작된 학술적으로 확립된 기법입니다.
- **SM-2 알고리즘**(SuperMemo, 1987)이 이미 컴퓨터 기반 간격 반복의 핵심 특허를 보유하고 있었으며(현재 만료), Anki를 포함한 수많은 구현체가 존재합니다.
- **FSRS(Free Spaced Repetition Scheduler)**: 2022년 이후 오픈소스로 공개된 최신 SRS 알고리즘으로, 머신러닝 기반의 고도화된 간격 예측을 제공합니다.

**비자명성 (Non-Obviousness): 낮음**

- 5단계 숙련도 체계는 SM-2의 ease factor 개념과 본질적으로 유사합니다.
- 관리자 UI에서 파라미터를 설정 가능하게 한 것은 "구성(Configuration) 기능"으로, 기술적 진보로 보기 어렵습니다.
- 체크 유형별 점수 차등 부여도 SM-2의 quality 등급(0~5) 개념의 변형입니다.

**선행기술 리스크 (Prior Art)**

| 선행기술 | 유사성 | 비고 |
|---------|--------|------|
| SM-2 알고리즘 (SuperMemo) | 간격 반복 핵심 알고리즘 | 특허 만료, 알고리즘 공개 |
| Anki SRS | 오픈소스 간격 반복 구현 | 광범위한 사용, GPL 라이선스 |
| FSRS (2022~) | 최신 SRS 알고리즘 | 오픈소스, MIT 라이선스 |
| Duolingo Birdbrain (내부 시스템) | AI 기반 학습 최적화 | 비공개이나 논문 존재 |
| Memrise SRS | 간격 반복 + 멀티미디어 | 상용 서비스 |
| US Patent 8,851,896 (Cerego) | 적응형 학습 시스템 | 유사 구조 |

### 3.3 권고: **출원 비권고 (Do Not File)**

- 간격 반복 알고리즘 분야는 **선행기술이 매우 풍부**하며, FlipFlip의 구현이 기존 기술 대비 충분한 기술적 진보를 보이지 않습니다.
- 관리자 UI에서의 파라미터 설정 기능은 "구성(Configuration)"에 해당하며, 특허 가능한 수준의 기술적 혁신으로 인정받기 어렵습니다.
- 단, 향후 **사용자 학습 패턴 데이터를 기반으로 개인별 망각곡선을 자동 조정**하는 AI/ML 기반 알고리즘을 개발한다면 재검토 가치가 있습니다.

## 4. 발명 후보 3: 가중 랜덤 응용문장 시스템

### 4.1 발명 개요

기본문장(Base Sentence)을 학습한 후, 해당 문장의 변형인 응용문장(Application Sentence)이 **가중 랜덤(Weighted Random)** 방식으로 출현하는 학습 시스템입니다.

```
[기본문장] "I go to school"
    ↓ 기본문장 숙련도 달성
[응용문장 풀] (가중 랜덤 선택)
  - "I go to school every day" (가중치: 높음)
  - "I used to go to school" (가중치: 중간)
  - "Do you go to school?" (가중치: 중간)
```

핵심 차별점:
1. 기본문장과 응용문장의 **명시적 참조 관계** (base_sentence_id)
2. 기본문장 숙련도 달성을 응용문장 출현의 **전제 조건(Prerequisite)**으로 설정
3. 응용문장 출현 시 **가중 랜덤** 적용 (단순 랜덤이 아님)

### 4.2 특허성 평가

**신규성 (Novelty): 중간**

- 기본→응용 문장의 계층적 학습 구조 자체는 교육학에서 일반적인 Scaffolding 기법입니다.
- 그러나 "기본문장 숙련도 달성 → 응용문장 가중 랜덤 출현"이라는 **구체적 트리거 메커니즘**은 기존 언어학습 특허에서 명확히 확인되지 않습니다.
- Prerequisite 기반 콘텐츠 해금은 게이미피케이션에서 일반적이나, 가중 랜덤과의 결합은 차별화 요소입니다.

**비자명성 (Non-Obviousness): 중간**

- "기본 학습 완료 후 심화 학습 제공"은 교육학의 기본 원리 (Zone of Proximal Development)
- 그러나 가중 랜덤을 통해 **학습 다양성과 반복 빈도를 동시에 최적화**하는 기술적 구현은 자명하지 않을 수 있음
- 가중치 결정 기준(숙련도, 유사도, 난이도 등)이 구체적으로 정의되면 비자명성이 강화됨

**선행기술 리스크 (Prior Art)**

| 선행기술 | 유사성 | 차이점 |
|---------|--------|--------|
| 교육학 Scaffolding 이론 | 단계적 난이도 상승 | 기술적 구현 아님 |
| Duolingo 문장 변형 | 같은 스킬 내 문장 변형 | 명시적 기본/응용 참조 관계 없음 |
| Adaptive Learning 특허 다수 | 학습자 수준 기반 콘텐츠 조절 | 가중 랜덤 메커니즘 상이 |
| US Patent 10,049,590 (Adaptive Learning) | 적응형 콘텐츠 제공 | 기본-응용 참조 구조 아님 |

### 4.3 권고: **추가 조사 필요 (Needs Further Research)**

- 클레임의 핵심을 "기본문장 숙련도 전제조건 + 가중 랜덤 응용문장 출현 메커니즘"으로 좁히면 등록 가능성 존재
- **가중치 결정 알고리즘**을 구체적으로 정의하면 비자명성이 강화됨 (예: 사용자 오답 패턴 기반 가중치 동적 조정)
- 한국(KIPO) 출원이 미국(USPTO) 대비 등록 가능성이 높을 수 있음 (한국은 소프트웨어 특허에 상대적으로 관대)
- **비용**: 한국 출원 약 300~500만원, 미국 동시 출원 시 추가 500~800만원. 선행기술 조사 먼저 권고.

## 5. 발명 후보 4: 레이더차트 숙련도 분석

### 5.1 발명 개요

사용자의 언어 숙련도를 **스킬 레벨 유형(시제, 문형)과 문장 레벨 유형(용도, 톤, 상황)**을 결합한 **다차원 레이더차트**로 시각화하는 분석 시스템입니다.

```
[레이더차트 축 구성]
  스킬 레벨 기반:          문장 레벨 기반:
  - 현재시제 숙련도         - 질문 용도 숙련도
  - 과거시제 숙련도         - 공식 톤 숙련도
  - SVO 문형 숙련도        - 비공식 톤 숙련도
  - 의문문 숙련도           - 비즈니스 상황 숙련도
  ...                      ...
```

핵심 차별점:
1. **이원 계층 데이터 소스**: 스킬 레벨 유형과 문장 레벨 유형을 모두 포함한 통합 분석
2. **원어민 수준 대비 시각화**: 각 축의 최대값이 원어민 수준(Level 100)으로, 사용자 현재 위치를 상대적으로 표시
3. **하이브리드 유형 시스템과의 연동**: 발명 후보 1의 데이터 구조를 입력으로 활용

### 5.2 특허성 평가

**신규성 (Novelty): 낮음**

- 레이더차트(Spider Chart/Radar Chart)를 이용한 다차원 능력 시각화는 **매우 일반적인 기법**입니다.
- 언어학습 분야에서도 CEFR 기반 능력 시각화, Duolingo의 스킬별 진행도 표시 등 유사 선행 사례가 다수 존재합니다.
- ETS(TOEFL/TOEIC)의 스코어 분석 리포트, Cambridge English의 능력 프로필 등 표준화된 다차원 분석이 이미 보편적입니다.

**비자명성 (Non-Obviousness): 낮음**

- 여러 차원의 데이터를 레이더차트로 시각화하는 것은 데이터 시각화의 기본 기법입니다.
- 스킬 레벨 유형과 문장 레벨 유형을 결합하여 축을 구성하는 것은 "두 데이터 소스를 하나의 차트에 표시"하는 것으로, 기술적 진보로 인정받기 어렵습니다.
- "하이브리드 유형 시스템에서 파생된 데이터를 시각화"한다는 점은 발명 후보 1에 종속적이므로, 독립적 특허 가치가 낮습니다.

**선행기술 리스크 (Prior Art)**

| 선행기술 | 유사성 | 비고 |
|---------|--------|------|
| CEFR 능력 프로필 | 다차원 언어 능력 시각화 | 국제 표준 프레임워크 |
| Duolingo 스킬 진행도 | 스킬별 숙련도 표시 | 상용 서비스 |
| TOEFL/TOEIC 스코어 리포트 | 영역별 점수 분석 | 수십 년간 사용 |
| 다수의 BI/Analytics 특허 | 레이더차트 기반 데이터 시각화 | 범용 기술 |

### 5.3 권고: **출원 비권고 (Do Not File)**

- 레이더차트 기반 숙련도 시각화는 선행기술이 매우 풍부하며, FlipFlip의 구현이 특허 가능한 수준의 기술적 혁신을 포함하지 않습니다.
- 하이브리드 유형 시스템(발명 후보 1)의 데이터를 시각화하는 방법으로서 가치가 있으나, 시각화 방법 자체는 특허 대상으로 부적합합니다.
- 단, 발명 후보 1이 특허 등록될 경우, 해당 특허의 종속항(Dependent Claim)으로 "하이브리드 유형 데이터의 레이더차트 시각화 방법"을 포함시키는 것은 가능합니다.

## 7. 결론

FlipFlip의 4가지 발명 후보 중, **하이브리드 스킬-유형 시스템(후보 1)**과 **가중 랜덤 응용문장 시스템(후보 3)**이 상대적으로 높은 특허 가능성을 보이나, 출원 전 선행기술 정밀 조사가 필수적입니다.

**망각곡선 알고리즘(후보 2)**과 **레이더차트 분석(후보 4)**은 선행기술이 풍부하여 출원을 권고하지 않습니다.

IP 보호 전략은 특허에만 의존하지 않고, **영업비밀 + 저작권 + 상표권 + 디자인 특허의 복합적 접근**이 비용 효율적입니다. 특히 2,680문장/13,400표현에 달하는 학습 콘텐츠는 저작권으로 충분히 보호 가능하며, 이는 실질적으로 가장 강력한 진입 장벽이 될 수 있습니다.
